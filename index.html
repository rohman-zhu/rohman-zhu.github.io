RDSH文件目录设置方案

Posted on 2026-07-10 In Windows Server

需求描述

期望用户在同一个Windows服务器中，仅能看到自己的文件夹，不允许看到其他人的目录；

实现方案

通过NTFS文件系统的安全配置；

设置方式

目标路径： D:/WorkSpace/

总的配置方案：每个用户首次登录时为其创建一个文件夹，并限制用户仅访问自己的文件夹。

NTFS 权限配置

  1. Administrators组 , 范围是” 应用于此文件夹、子文件夹和文件 (This folder, subfolders and files)” ，权限是 “完全控制 (Full Control)” ；
  2. 对于最终用户：
  • Users组，范围”仅此文件夹 (This folder only)”，权限”创建文件夹/附加数据 (Create folders / append data)”;
  • Remote Desktop Users组，范围”仅此文件夹 (This folder only)”，权限为”创建文件夹/附加数据 (Create folders / append data)”;
  1. Creator Owner组(创建者所有者)，范围”仅文件夹和文件 (Subfolders and files only)”,权限是”完全控制 (Full Control)”。

ESXi上线前的Checklist记录

Posted on 2026-06-27 In default

ESXi提示其他主机硬件对象的状态告警：H3C R4900 G3

Posted on 2026-06-27 In VMware , vSphere

故障现象

在ESXI的监控，硬件运行状况，警示和警告显示两条：Connectivity Switch 36 LAN_4_Link 0和Connectivity Switch 36 LAN_3_Link 0，但是登陆H3C R4900 G3，进去HDM，却无任何告警。
ESXi主机的上行链路端口正常；

原因分析

可能是板载网卡没有接网线，导致ESXi层识别到异常；

解决方法

  1. 进入HDM（H3C的带外）；
  2. 进入 HDM设置，网络设置，共享网口，取消ipv4\ipv6的配置或取消勾选 LAN设置；
  3. 重启HDM；
  4. 进入ESXi，刷新健康日志；

记一次vscode更新后的问题

Posted on 2026-05-30 In default

故障现象

VSCode 在一次自动更新后，无法通过vscode连接到linux服务器做一些编辑工作，每次通过vscode连接服务器时，在输入密码后均会卡在 setting up sshd host xxx:downloading vs code server；

原因分析

应该是组件更新后，需要在目标服务器中安装对应插件；由vscode远程连接linux服务器时，服务器会触发wget指令

服务器解析update.code.visualstudio.com域名，得到的IP地址有异常，导致无法正常通讯；

解决方法

在linux服务器中添加 update.code.visualstudio.com 的解析记录至本地 hosts文件；
