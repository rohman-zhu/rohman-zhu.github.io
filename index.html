旁路由

旁路由其实并不是路由，路由是用来连接不同网络的，最常用的就是用来连接互联网和局域网。旁路由起到的主要是网关的作用，是用来分流数据和扩展插件的。因此，严谨一点的叫法应该是 旁路网关，只是大家好像约定俗成了都叫做旁路由，所以我们这里也跟着叫旁路由，但是要明白它的核心是网关而不是路由。

要实现分流的常规做法是改造现有网络架构，修改主路由配置；但“旁路由”模式可以不折腾现有的网络架构，只是在链路上挂载一个组件即可实现网络访问的分流；

常规拓扑如下：

graph LR

主路由-->外网
主路由-->旁路由

常规的做法是需要部署DHCP服务器，在DHCP服务器中对特定设备的默认网关做区分，常规设备就使用正常的网关，无法走代理流量；而特殊的设备，默认网关则调整为旁路由的IP；

如何获取$Credential

使用加密的密码文件

这种方式是先输入密码生存一个加密文件，该加密文件仅在当前账户与当前服务器下登录才可用；

ConvertFrom-SecureString 与 ConvertTo-SecureString用途

  1. SecureString:PowerShell 中的一种特殊字符串类型，它在内存中对数据进行加密，避免以纯文本形式暴露敏感信息（如密码）。当你使用 Read-Host -AsSecureString 或 ConvertTo-SecureString 时，就会得到 SecureString 对象。
  2. ConvertFrom-SecureString: 将这个 SecureString 对象转换成一个普通的字符串（实际是加密后的文本），然后你就可以将它保存到文本文件、数据库或环境变量中。

默认情况下，ConvertFrom-SecureString使用 Windows 数据保护API(Data protect API)进行加密:

  1. 加密后的字符串只能由同一台计算机上的同一个用户账户解密。
  2. 如果你尝试在另一台计算机或以不同用户身份运行解密（使用 ConvertTo-SecureString），操作将会失败。
  3. 这种机制确保了即使密码文件泄露，攻击者也无法在其他环境下使用它。

.plist文件编写

launchd是通过“.plist”后缀结尾的xml文件来自定义程序启动

  1. Label为需要启动的进程设置一个标签，后续使用launchctl load [服务名]中的服务名对应的是这个Label后面的string标签里面的node_exporter；
  2. ProgramArguments为程序运行时所带的参数；
  3. RunAtLoad表示在加载时启动，后面跟的参数如果是则表示launchd加载时就自动启动，如果是则需要手动启动；
  4. KeepAlive参数表示是否要一直存活，如果后面参数是则表示会持续启动，每个10s启动一次；
