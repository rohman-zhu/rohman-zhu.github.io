ESXi上线前的Checklist记录

Posted on 2026-06-27 In default

ESXi提示其他主机硬件对象的状态告警：H3C R4900 G3

Posted on 2026-06-27 In VMware , vSphere

故障现象

在ESXI的监控，硬件运行状况，警示和警告显示两条：Connectivity Switch 36 LAN_4_Link 0和Connectivity Switch 36 LAN_3_Link 0，但是登陆H3C R4900 G3，进去HDM，却无任何告警。
ESXi主机的上行链路端口正常；

原因分析

可能是板载网卡没有接网线，导致ESXi层识别到异常；

解决方法

  1. 进入HDM（H3C的带外）；
  2. 进入 HDM设置，网络设置，共享网口，取消ipv4\ipv6的配置或取消勾选 LAN设置；
  3. 重启HDM；
  4. 进入ESXi，刷新健康日志；

故障描述

SSH无法登录，且无法从控制台登录（tty） ， 现象是 输入密码回车后，界面会短暂卡住，然后复原回去，提示输入账户名与密码。

原因分析

用户尝试登录 –> PAM 认证通过(密码正确) –> PAM请求 DBus 激活 systemd-logind –> DBUS提示“Refusing activation” –> systemd-logind 无法创建 Session –> 登录失败

佐证材料

dbus日志

DBus 为什么会 shutting down?

解决方法

临时方案：重启服务器大法

重启服务器 –> DBus/systemd-logind 重新初始化 –> 恢复

记一次vscode更新后的问题

Posted on 2026-05-30 In default

故障现象

VSCode 在一次自动更新后，无法通过vscode连接到linux服务器做一些编辑工作，每次通过vscode连接服务器时，在输入密码后均会卡在 setting up sshd host xxx:downloading vs code server；

原因分析

应该是组件更新后，需要在目标服务器中安装对应插件；由vscode远程连接linux服务器时，服务器会触发wget指令

服务器解析update.code.visualstudio.com域名，得到的IP地址有异常，导致无法正常通讯；

解决方法

在linux服务器中添加 update.code.visualstudio.com 的解析记录至本地 hosts文件；
