现象说明

所以核心诉求1实现了，但核心诉求2却无法实现；

E810网卡踩坑记录-RHEL不兼容问题

Posted on 2025-10-15 In 服务器与存储

故障现象

物理服务器安装RHEL7.9版本，有以下故障现象：

  1. 服务器网卡不识别；
  2. 安装网卡驱动，无法安装；
  3. 安装网卡驱动后，服务器配置IP不生效，ip指令查询，状态为UP，但有NO-CARRIER字样；
  4. 服务器卡死；
  5. 关闭超线程后，服务器不会卡死；

原因分析

对于基于 AMD Rome CPU 的系统，当逻辑 CPU 的数量大于 124 个时，其上的 E810 NIC 接口无法正常工作。

临时解决方案

可以通过传递 nr_cpus=124 到内核命令行；屏蔽可用CPU核数；

永久解决方案

此问题已在 RHEL 7.9 z-stream 内核版本 kernel-3.10.0-1160.21.1.el7.x86_64 中得到修复。

应急响应预案-远程办公支持

Posted on 2025-09-24 Edited on 2025-10-22 In VMware , Horizon View

背景

遇到突发事件导致员工需要远程办公，远程办公的方式为
mstsc –> 内网PC

可以采用 VMware Horizon View发布虚拟应用；

需要关注的节点与问题

角色 关注点 解决方案
安全服务器 或 UAG 计算资源与网卡是否能支撑现有用户规模；因承担用户流量第一道关口，需要做流量转发，因此需要关注CPU、网卡性能；但内存也需要关注，避免内存泄漏； 配置改为8C32GB，网卡支持1Gbps （参照200人规模的远程桌面并发：CPU 30%，内存 10%，网络流量可达200Mbps，入站100Mbps，出站100Mbps）
连接服务器 关注用户数,因为是windows server，不能出现内存溢出，导致服务崩溃 8C/16GB即可
RDSH服务器 关注资源负载，不要让CPU、RAM使用率超过50%，磁盘空间不要用满；关注用户数量（尽量控制在60以内，30个最优） 配置为16C/64GB即可（54人并发，32C/128GB,CPU负载33%，内存24%，网络45Mbps）

核心：用户数 + 负载；

监控方案

安全服务器

  1. CPU、RAM、Storage、网络流量：zabbix 默认的windows模板即可；
  2. 连接数监控：8443

做成脚本,设置定时任务，输出成Json格式，格式为{“date”:”2025-09-24”,”connected_8443_count”:”2”};

连接服务器

  1. CPU、RAM、Storage、网络流量：zabbix 默认的windows模板即可；
  2. 用户数监控：horizon REST API ;

RDSH服务器

  1. CPU、RAM、Storage、网络流量：zabbix 默认的windows模板即可；

应急开始，快速扩容

RDSH 横向扩容方案

拿到机器后，开始做初始化，让服务器具备远程控制能力；然后再从统一节点下发远程安装指令；

步骤一、快速初始化；

步骤二、在控制节点下发远程部署指令，如安装agent、zabbix之类的操作；

安全服务器、连接服务器扩容

服务器可以通过负载均衡，临时停用需扩容的节点，不让用户流量从“需变更的节点”经过；

  1. 切换流量停用节点；
  2. 服务器关机纵向扩容；
  3. 开机后验证业务；
  4. 确认无误，启用节点；

应急结束，回收资源

RDSH回收

  1. 停用节点，确保无用户在用，关闭RDSH；
  2. Horizon Admin界面从场中移除目标RDSH；
  3. Horizon Admin界面取消注册 目标RDSH；

安全服务器、连接服务器资源回收

  1. 切换流量停用节点；
  2. 服务器关机缩容；
  3. 开机后验证业务；
  4. 确认无误，启用节点；

IOS17不完美降级-iPad6降级记录

Posted on 2025-09-21 In default

背景

2018年买的iPad 6（型号：A1893），在25年不小心由IOS14更新至IOS17，使用起来非常卡，日常看B站视频都会卡顿，于是我一直在寻找方法降级，哪怕不完美降级也好；
终于在网上找到 turdus 降级方案，中文互联网有人说是 乌鸦 或者 乌鸫 本质上都是同一个东西，在这方面有一个UP主做的比较全面，把工具包集成到同一个脚本工具内，并免费发布；（小红书上很多人靠这个吃饭，不完美降级一次 95块，连工具包的名字都不改…）

前置条件：

  1. 待刷入的固件下载：来自爱思助手；
  2. 工具包：玄烨苹果（https://dkxuanye.cn/?p=11303），UP主已经封装了工具包为1.0.4版本；
  3. 设备准备：一台Intel的MacBook（核心是要Intel+MACOS） 、 一根稳健的Light数据线、一台待刷机的 iPad；
  4. 学习 iPad 怎么进入DFU模式；
  5. 提前将 iPad 的数据做好备份；（数据无价，如果有重要数据一定要提前备份）

刷机流程

  1. 确定设备是否在turdus merula的支持范围内，A9、A10芯片的设备才支持；
  2. 下载对应设备的固件，将固件放置在工具包目录中；
  3. 确定设备完成数据备份后，将设备进入 DFU模式 模式，准备刷机；
  4. 将设备连接PC，执行工具包，执行刷机；
  5. 刷机完成后，再次将设备进入DFU模式，并执行工具包，进行引导开机；

MAC OS下的刷机实操

  1. MAC电脑要关闭系统完整性保护，在Mac的Recovery模式下，打开终端，输入 csrutil disable ；【这个东西会影响软件运行，可能会导致工具的一些脚本无法运行，参照https://sspai.com/post/55066】
  2. 设备先从爱思助手进入恢复模式，再进入DFU模式，DFU模式需要自己手动操作；【核心还是需要进入DFU模式】
  3. 下载固件包，放置到软件工具的目录下；【这个要将固件与工具一起存放才可以运行工具】
  4. 运行软件Dk_turdus merula，是一；
  5. 输入6;【“开始刷机降级”】
  6. 提示SEP和Baseband的时候，输入1，回车；
  7. 清空手机数据提示时，输入YES，回车；
  8. 将设备进入DFU模式；
  9. 接入MAC，打开软件，输入7，开始引导；
  10. 正常引导；
