MACOS的监控方案-node_exporter安装与部署

Posted on 2026-02-12 In default

需求背景

需要监控MACOS的信息，如CPU、RAM、Storage的使用率信息；还需获取到MACOS 内的资产数据信息，比如CPU型号、RAM配置、序列号、OS版本；拟定采用的方案是Prometheus，需要在MAC OS中安装 node_exporter，通过Prometheus Server采集数据，并由Grafana做数据展示；

实施实例

node_exporter 安装

darwin是MACOS的操作系统内核与底层核心环境；下载包中的darwin表示为macos版本；

node_exporter启动与开机自启

node_exporter直接执行是一个前台进程，访问时会打印出日志；一般是将此服务做成守护进程或者放置到docker中；

设置开机自启，则需要注册成守护进程，在 /Library/LaunchDaemon中编写plist文件，并注册；

MAC OS的守护进程

我们需要将进程放置到后台自动运行或者开机时程序能自动启动；
这里LaunchAgent与LaunchDaemon是有差异的，Agent是需要用户登录后才能启动，Daemon则是随机器启动；

详情可以参考：
https://developer.apple.com/library/archive/documentation/MacOSX/Conceptual/BPSystemStartup/Chapters/CreatingLaunchdJobs.html
https://developer.apple.com/library/archive/technotes/tn2083/_index.html#//apple_ref/doc/uid/DTS10003794

.plist文件编写

launchd是通过“.plist”后缀结尾的xml文件来自定义程序启动

  1. Label为需要启动的进程设置一个标签，后续使用launchctl load [服务名]中的服务名对应的是这个Label后面的string标签里面的node_exporter；
  2. ProgramArguments为程序运行时所带的参数；
  3. RunAtLoad表示在加载时启动，后面跟的参数如果是则表示launchd加载时就自动启动，如果是则需要手动启动；
  4. KeepAlive参数表示是否要一直存活，如果后面参数是则表示会持续启动，每个10s启动一次；

2026-01-20记录：注册成功后有效期1年，到期后180天内可以免费续订；

查询生效与否

一定要开启”Enable privacy”，会隐去手机号、住址信息，仅保留邮箱地址；

DNS 注册

使用的是赛博菩萨，cloudflare ，选择CloudFlare原因：

  1. 注册CloudFlare账户；
  2. 在CloudFlare添加域名，选择免费计划；
  3. CloudFlare会提供两个DNS服务器，比如daisy.ns.cloudflare.com,fred.ns.cloudflare.com;将这两个DNS服务器添加到DigitalPlat中创建的域名的Nameserver configuration里面；

如果代理状态不激活，则返回的是真实源服务器IP；如果启动了代理状态，则返回的是CloudFlare内部的服务器IP，但这里有问题的是，启用CloudFlare的代理状态，则无法跟目标服务器通讯，还待查明问题；

query_history_k_data_plus() 参数说明

  • code：股票代码，sh或sz.+6位数字代码，或者指数代码，如：sh.601398。sh：上海；sz：深圳。此参数不可为空；
  • fields：指示简称，支持多指标输入，以半角逗号分隔，填写内容作为返回类型的列。详细指标列表见历史行情指标参数章节，日线与分钟线参数不同。此参数不可为空；

  • frequency：数据类型，默认为d，日k线；d=日k线、w=周、m=月、5=5分钟、15=15分钟、30=30分钟、60=60分钟k线数据，不区分大小写；指数没有分钟线数据；周线每周最后一个交易日才可以获取，月线每月最后一个交易日才可以获取。
  • adjustflag：复权类型，默认不复权：3；1：后复权；2：前复权。已支持分钟线、日线、周线、月线前后复权。 BaoStock提供的是涨跌幅复权算法复权因子，具体介绍见：BaoStock复权因子简介。

Grafana学习

Posted on 2025-12-08 In Linux

背景

学习一下Grafana+Json的监控展示方案；

环境说明

本次使用的环境用的是WSL + Grafana，WSL用的是ubuntu24.04；

配置软件仓库

Ubuntu 24.04里面需要配置

安装marcusolsson-json-datasource插件

在左边导航栏中，Connections/Add new connection里面，搜索marcusolsson-json-datasource，查询到JSON API；

点击JSON API，点击“Install”

配置JSON数据源

可以将Json数据放入 /usr/share/grafana/public 目录下；

本次将json数据放入 /usr/share/grafana/public/data.json;
对应的路径就是 http://localhost:3000/public/data.json;

添加数据源:

  1. Connections/Data sources，点击“Add data source”;
  2. 点击“JSON API”；
  3. Name随便填写
  4. 关键的是Connection里面的URL，以下有两种方式；

导入方式一：json文件的绝对路径读取

添加数据源

在data source里面，添加JSON API，里面的URL填写的是 http://localhost:3000/public/data.json ;
点击“Save & test”

创建面板
  1. 点击 Dashboards，点击“Create dashboard”；
  2. 点击“Add visualization”;
  3. 选择刚才创建的 “data source”;
添加查询

因为刚才添加的是绝对路径的数据源，而我们的数据就是标准的json格式，所以在Fields中，我们就可以填入对应的数据了；

  1. 日期数据：$[*].date;
  2. 具体数据：$[*].mortgage.mortgage_interest;
添加面板样式
  1. 右边的 Visualization 中，选择 Table，则可以获取到表格样式；
  2. 右边的 Visualization 中，选择 Time series，则可以得到则线图；

导入方式二：json文件的相对路径

添加数据源

在data source里面，添加JSON API，里面的URL填写的是 http://localhost:3000;
点击“Save & test”

创建面板
  1. 点击 Dashboards，点击“Create dashboard”；
  2. 点击“Add visualization”;
  3. 选择刚才创建的 “data source”;
添加查询

在下方的Path选项卡，填入/public/data.json ，并选择GET方法；

因为刚才添加的是绝对路径的数据源，而我们的数据就是标准的json格式，所以在Fields中，我们就可以填入对应的数据了；

  1. 日期数据：$[*].Date;
  2. 具体数据：$[*].mortgage.mortgage_interest;
添加面板样式
  1. 右边的 Visualization 中，选择 Table，则可以获取到表格样式；
  2. 右边的 Visualization 中，选择 Time series，则可以得到则线图；
