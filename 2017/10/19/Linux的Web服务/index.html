Web

网站访问俗语

IP-Internet_Protocol

Internet-Protocol (独立IP),独立IP数是指不同IP地址的计算机访问网站的被计算的总次数. (以一天为计算)

一个客户机通过ADSL拨号上网,但上网偶尔掉线,一共重新拨号三次,然后去访问百度,此时百度的独立IP数有多少

答案: 3次.

PV-Page View

页面浏览,即页面浏览量或点击量,不管客户端是不是相同,也不管IP是否相同,用户每次访问一个网站都会被计算一个PV.

一个来访者访问网站,PVC数可能产生若干,但是独立IP数就为1个,因此如果对比一个网站的独立IP数和PV数,PV数一定会大于独立IP数,视网站的业务而定,例如这种分类门户可能要达到10:1,甚至更多.

UV-Unique Visitor

同一台客户端(PC或者移动端)访问网站被计算为一个方可.

考虑到一台客户端电脑可能会被多人使用,因此UV实际上并不一定是独立的自然人访问.

UV度量

  1. 通过客户端HTTP请求报文分析 .(用IP地址+其他特征共同定义)
  2. 通过Cookie.

统计

  1. 分析 Web 服务器的访问日志信息,对IP地址段去重后统计.
  2. 第三方统计,谷歌的统计(GA)
  3. 在网站的每一个页尾嵌入js等统计程序代码,当用户加载网页后,发给统计IP的服务器.

DNS解析基本流程

Domain Name System

常用:

  • 设置 CNAME 别名记录
  • 设置 MX 邮件记录
  • 设置 PTR 记录 , 方向解析, 即把IP地址解析为对应的域名.

基本流程

  1. 用户查找本地 Hosts 文件(本地DNS缓存)
  2. 继续请求客户机DNS服务器
  3. 请求全球 DNS 服务器

HTTP协议

HyperText Transfer Protocol,超文本传输协议.属于OSI模型中的第七层协议.


WWW(World Wid Web),默认的服务端口是80,另一个加密的www服务(HTTPS)默认服务端口是443,用于网银,支付等.


有这么几个主要的版本:
  • HTTP 0.9
  • HTTP 1.0
  • HTTP 1.1

推荐的书籍: HTTP权威指南(前4章)

1.1支持持久连接,在一个TCP连接上可以传送多个HTTP请求和响应.

HTTP 请求方法

每一个HTTP请求保温中都包含一个方法,例如:获取界面,提交内容到服务器.删除资源.

HTTP方法 作用
GET 客户端请求指定资源信息,服务器范返回指定资源
HEAD 只请求响应报文中的HTTP 首部
POST 将客户端的数据提交到服务器
PUT 从客户端向服务器传送的数据取代指定的文档内容
DELETE 请求服务器删除指定的资源
MOVE 请求服务器指定的页面移至另一个网络地址

HTTP 状态码

状态码 作用
100-199 指定客户端应相对的某些动作
200-299 用于表示请求成功
300-399 用于已经移动的文件并且常被包含在定位头信息中指定新的地址信息
400-499 用于指定客户端的错误
500-599 用于指出服务器错误
  • 200 表示成功
  • 301 表示永久跳转,所有请求的网页将永久跳转到被设定的新位置
  • 403 禁止访问
  • 404 服务器找不到客户端请求的指定页面
  • 500 内部服务器错误,例如SELinux开启,而又没有为http设置许可规则
  • 502 (Bad Gateway)网关故障,基本跟代理有关
  • 503服务当前不可用 服务当前不可用,可能因为服务器超载或者维护
  • 504 网关超时,一般是网关代理服务器请求后端服务时,后端服务没有在特定的时间内完成处理请求

协议:分隔符号:IP地址:分隔符号:资源目录地址

  • Query Per Second - 查询
  • IOPS(Input/Output Oeperations Per Second) - 硬盘读写

问题

什么是静态资源 ? 作用以及优缺点是什么

URL固定,后缀是HTML,HTM,服务器存在的文件.

特点: 无可交互的数据库

  • 优点: 性能高,体验好 ; 便于SEO(Search Engine Optimizatio , 网站推广)
  • 缺点: 交互差;维护麻烦

什么是动态资源?作用以及优缺点

有数据库支持,内容丰富 ,后缀是.php , jsp ,asp , aspx

特点:

  • 优点: 交互性好(登录,注册);维护简单;
  • 缺点: 性能差(服务器端解析) ; URL变化不利于搜索

什么是伪静态?作用以及缺点

动态URL伪装成静态,rewrite规则模块

  • 优点 : 方便收录 ; 提升用户浏览体验
  • 缺点 : 不会提升网站性能,微降
