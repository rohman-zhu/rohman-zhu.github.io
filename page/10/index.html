BIOS设置与CPU温度问题

Posted on 2023-01-02 Edited on 2024-12-22 In 服务器

问题描述

服务器CPU的型号为 Intel Xeon Gold 6254 (基础频率为 3.1 GHz ， 睿频可达 4.0 GHz);
产品信息 https://ark.intel.com/content/www/cn/zh/ark/products/192451/intel-xeon-gold-6254-processor-24-75m-cache-3-10-ghz.html

OS为 Ubuntu 18.04，OS内用指令 lscpu、 cpupower frequency-info 查看到CPU的最大主频为3.2GHz ，而没有达到 4.0 GHz；

原因分析

  1. BIOS没有开启CPU的睿频模式，也就是Turbo Mode；
  2. OS内禁用CPU的PState ，进而禁用睿频；

解决方法

BIOS设置

进入BIOS ， Processor Configuration / Advanced Power Management Configuration

OS内的设置

Grub启动参数不能将 pstat 状态禁用；

问题描述

服务器没有运行任务（空载）的情况下，CPU温度达到70℃，甚至会达到82℃。（OS中断的温度104℃，此CPU最大允许的运行温度是82℃；）

看到系统在空载的情况下，CPU当前主频会达到3.9GHz 。并持续在3.9GHz；

BMC将十个散热风扇调整到最大转速，依旧无法有效降温，CPU温度轻松突破82℃。

原因分析

相当于CPU在满性能运行，要找出让CPU满性能运行的原因。

  1. BIOS将CPU的C State调整成最高性能模式，并持续为高性能模式；
  2. OS内手动将性能调整至最高性能模式；

解决方法

BIOS设置

将CPU的C State模式调整为Auto

OS 检测

一般为Grub菜单初始化

远程桌面RDP负载记录

Posted on 2022-12-27 Edited on 2024-12-22 In Windows Server

业务描述

使用MSTSC（远程桌面）工具到其他PC，也就是基于RDP协议去远程。
一台RDSH（Remote Desktop Session Host）配置为16C32G，单核主频为2.49GHz 。

业务主要特征：

  • 与本地显示器的分辨率有关，本地显示器的分辨率越高，RDSH对CPU的要求就越高；
  • 跟远程投射的画面变化量有关，屏幕变化量越大，RDSH对CPU的要求就越高；

业务负载记录

资源机RDSH记录

XX月XX日 15:04

  • 用户数：22
  • CPU负载：90%
  • RAM负载：50%

描述：远程会有卡顿感。如果远程到目标端是多层RDP嵌套，则会更加卡顿。

安全服务器记录

安全服务器配置4C8G；

XX月XX日 16:03

新增加RDSH后，卡顿感明显；检查链路上的前端节点。

  • 用户总数：220
  • CPU负载：100%
  • RAM负载：50%
  • 网络流量负载：80Mbps；

描述：用户数量达到 200 左右时， 安全服务器节点的CPU应该扩容至8C。

xx月xx日 16:36

安全服务器配置为8C16G；

  • 用户总数：241
  • CPU负载：
  • RAM负载：50%
  • 网络流量负载：

改进措施

用户侧

要对用户做分类，根据用户实际业务场景对CPU使用情况由高到低分为重型用户、轻型用户、常规用户。

重型用户在申请使用的环节就要识别出来，并把重型用户要归到高配的RDSH，避免其工作影响其他轻型用户、常规用户使用；

资源侧

  1. 优化监控项；
  2. 优化部署流程；

优化监控项

主要的问题：

  1. 未能及时识别到告警的节点在链路中承担的角色，例如：告警信息只有IP信息，不清楚其为重要的安全服务器角色。
  2. 告警等级不准确，例如：该为2级告警的信息被定义为3级告警。
  3. 监控信息查看不方便，没有做信息聚合，例如：需要查看CPU使用率与用户数结合。
  4. 没有办法及时评估整体负载情况。

记录一次CPU超线程导致NVME固态盘故障【待补充】

Posted on 2022-12-20 Edited on 2024-12-22 In 服务器与存储 , 服务器

故障描述

有一批相同配置的服务器，大部分服务器上面的磁盘读写始终不及预期。原本以为这就是服务器所能达到的实际速度，然而，这一批次中却有一台服务器速率却能达标。因此记录此次故障记录，避免后续再遇到。

服务器大致配置，关键配件为CPU与NVME SSD；

原因分析

服务器使用两个AMD的CPU，单颗CPU为64Core，总共是128Core；而NVME SSD的队列为128，如果开了超线程，则会导致CPU有256Core去与NVME交互，会导致SSD的一个队列在等待2个Core的调度（争用），从而导致整体效率下降。【猜想】

解决方法

关闭超线程
