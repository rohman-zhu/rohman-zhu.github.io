指定DNS解析特定域名

Posted on 2021-02-22 Edited on 2024-12-22 In Linux

背景

有一套存储是通过一个域名来对外提供服务，客户端需要通过域名来连接存储。

虚拟机需要解析这个制定域名才能实现挂载存储，而这个域名需要使用存储自带的DNS才可以。
然而存储的DNS是在其局域网内，存储的DNS与业务网络并不相通，因此业务网络中的DNS无法实现转发DNS请求到存储的DNS上。

因此虚拟机有这两个要求：

  1. 正常情况下还是需要使用业务网络的DNS，即非存储的域名解析都要在业务网络DNS中执行。
  2. 存储的域名解析则需要在存储的DNS中执行。

方案

方案一

在Linux的DNS配置中增加轮询机制，使存储的域名解析请求到制定DNS。

缺点：这种方式也会让正常业务的DNS解析请求到存储的DNS上，其次存储的域名解析还是有概率到正常业务网络的DNS上。

方案二

在Linux中安装dnsmasq，这个软件可以制定dns解析请求。

方案三

业务网络与存储网络的DNS相通，则在业务网络的DNS中配置DNS转发请求。

记一次ESXI管理端故障

Posted on 2021-02-22 Edited on 2024-12-22 In VMware , vSphere

故障现象

vCenter已经无法管理ESXI，并且无法单独连接此ESXI。
ESXI上的业务还在正常运行，应该是管理进程异常。

原因分析

可能是ESXI的Management Agent 异常，可以通过ESXI的console去重启Management Agent。

操作流程

为了避免重启Management Agent时，ESXI 卡在 “Stopping ESXI Management Agent”界面，因此需要提前打开 ESXI Shell 和 SSH 。

  1. 从物理服务器的带外KVM进入ESXI，开启ESXI Shell 与 SSH。
  2. 重启 Management Agent 。

如果出现异常，可以通过 ssh 或者 ESXI Shell 中断重启进程。

如果以上方法无效，可以采用以下方法，先迁移业务：
（此时业务已经无法通过vCenter去管控，必须人为干预，重新注册虚拟机。）

  1. 先记录故障ESXI上的虚拟机，包含 IP 、 虚拟机名 、 底层所用的Datastore 。
  2. 从vCenter中移除未响应的ESXI（因为要重新注册虚拟机，如果当前ESXI上的虚拟机还在注册状态，则无法执行注册指令。移除ESXI并不会影响ESXI上的业务。）
  3. 联系虚拟机对应的运维人员，约定时间关机后，找到虚拟机对应vmx文件，执行注册。
  4. 虚拟机执行开机，联系业务管理员检测业务。

记一次群辉NAS的iSCSI LUN故障

Posted on 2021-02-09 Edited on 2024-12-22 In default

背景

服务端通过iscsi协议挂在一个群辉设备的iSCSI卷。并把这个盘作为共享文件，客户端通过Samba协议连接到服务器，并将文件传输落盘至这个iSCSI卷。

故障现象

  1. 客户端传输速度变慢，多数时候为0kb/s;
  2. 当有传输任务发起时，客户端打开映射盘时会出现无响应的情况；
  3. 传输任务容易中断。

原因分析

  1. 网络原因1：客户端到服务端的网络链路。
  2. 网络原因2：服务器端到存储设备的网络链路。
  3. 服务器原因1：服务器存在异常进程或者服务。
  4. 服务器原因2：服务器负责较高。
  5. 存储服务器原因1：设备存在故障，如故障硬盘、阵列降级。
  6. 存储服务器原因2：所在的iSCSI LUN存在异常。

Windows-动态访问控制

Posted on 2021-01-31 Edited on 2024-12-22 In Windows Server

DAC动态访问控制

文件的访问控制是基于 NTFS 文件系统来执行的。因此要使用此特性，文件系统必须为NTFS。

NTFS 特性

  1. 节约空间
  2. 磁盘压缩
  3. 更加安全
  4. EFS加密
  5. 磁盘配额
  6. 卷影副本
  7. 单个文件最大支持64GB

访问控制

操作流程：

  1. DC中启用足额策略：计算机管理-系统-KDC-KDC支持声明、复合身份验证和Kerberos Armoring
  2. 声明属性:在动态访问控制中，创建声明，如department 、title
  3. 创建规则：在动态访问控制中，进入 Central Access Rules中，创建规则
  4. 创建策略：在动态访问控制中，进入 Central Access Policy 中添加规则
  5. 目标文件服务器，选中对应文件夹的属性，高级，中央策略，选择对应Policy

Claim Types 声明
Central Access Rules 规则
Central Access Policy 策略（包含多个规则）

Policy 通过组策略发布，在客户端生效。

“拒绝优先”

ACL列表中，依次从上至下执行，被列为拒绝权限的用户，优先执行。

需求场景

创建一个public的文件夹，仅允许公司内的主管访问。

传统做法

创建一个安全组，将主管的AD拖进同一个安全组，然后再对Public进行授权。

Windows活动目录学习

Posted on 2021-01-31 Edited on 2024-12-22 In Windows Server

基础概念

Item Detail
域 用来描述一种系统架构，和“工作组”相对应，由工作组升级二来的高级架构，在域架构中，可以实现统一化管理。
活动目录 微软提供的目录服务器（查询、身份验证），活动目录的核心包含了活动目录数据库，在活动目录数据库中包含了域中的所有对象（用户，计算机，组）
域控制器 在域架构中用来管理客户端的服务器，是域架构的核心，每个域控制器上包含了活动目录数据库。

组策略

Question：组策略中的设置“计算机管理”和“用户管理”管理对象分别是什么？
Answer：

  • 计算机管理，对应的是活动目录下的计算机
  • 用户管理，对应的是活动目录下的用户

DC部署流程

  1. 创建Windows Server，将服务器加入现有的域。（如果当前不存在域控，可忽略）
  2. 安装域控相关的服务器。（Active Directory 、 Domain Server等）
  3. 将服务器提升为域控制器。

注：

  1. 作为域控的服务器文件系统需要为NTFS。
  2. 不需要提前安装DNS服务，因为安装过程中会自动安装DNS服务。

域控问题

【功能级别】林功能级别和域功能级别是什么意思？

  • “林”功能级别，控制的是整个林中的域控制器的操作系统版本。
  • “域”功能级别，控制的是整个域中的域控制器的操作系统版本。

【域名申请】公网域名为rohman.cn ， 域名要为rohman.cn还是rohman.local？

如果域名使用rohman.cn ，公网上对外发布的服务器域名为crm.rohman.cn时，内网需要单独添加解析，否则会无法查询到此记录。

关于 netlogon 、 sysvol

sysvol ： 组策略模板文件脚本文件。
netlog

关于域是一个安全边界

Domain Detail
rohman.cn 父域
pz.rohman.cn 为rohman.cn的子域
sg.rohman.cn 为rohman.cn的子域

子域与父域是信任关系。

父域的管理员 rohman.cn/rohman.zhu 无法管理两个子域，仅有读取权限。

关于域是管理边界

关于域是一个复制单元

  1. 域中的DC的数据库是相互复制的（目的是为了确保多个副本）
  2. 同一个站点中，默认15s复制一次。不同站点的复制频率则取决于站点链路的频率。
  3. 复制类型有两种：多主复制、单主复制。

活动目录数据库有三个逻辑分区：Domain ， Configuration ， Schema ；同一个域中的DC，三个分区都会数据同步；如果不是同一个域，则只同步Configuration 、 Schema分区。

目录树与目录林

目录树类似于一个内家庭不同家庭成员分支。
目录林类似于一个家族内的不同家庭分支。

物理结构

站点表示的是一个物理区域。

站点

好处：可以让当地的服务器寻找当地的DC去验证。只有在本地站点出现故障的情况下，才会去其他站点做验证。

【站点同步】

默认触发：默认情况下，站点是180分钟发起一次同步。
手动触发：“Active Domain站点和服务” 选中对应站点的NTDS Setting选中链路，右键-立即复制

验证成功的前提条件：

  1. 输入DC中存在的账户，并且是正确的账户与密码。
  2. DC中能找到当前登录的计算机。
