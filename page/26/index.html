RD会话主机120天宽限期已过的临时解决方法

Posted on 2020-05-30 Edited on 2024-12-22 In Windows Server

故障描述

未配置远程桌面会话主机服务器的授权模式

原因分析

  1. 根本原因： 没有配置RD授权服务器的地址与授权模式。
  2. 直接原因： 临时License 是120天的宽限期，当前已过期。

临时解决方法

可尝试删除如下注册表

HKEY_LOCAL_MACHINE\Software\Microsoft\MSLicensing

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Terminal Server\RCM\Grace Period

如果无法直接删除，鼠标右键 Grace Period ，选择“权限”，点击“高级”，进“所有者” 选项卡，将所有者改成当前用户，并复选“替换子容器和对象的所有者”后，点击确定。

右键 Grace Period ，选择“权限” ， 添加everyone 权限后，将其删除。

Windows自动重启事件

Posted on 2020-05-30 Edited on 2024-12-22 In Windows Server

故障描述

服务器在凌晨2点左右自动重启了。

原因分析

  1. 打开事件查看器，看到有安装补丁的提示“将在15分钟内重启”
  2. 事件ID，1074。
  3. 在C:\Windows\下的windowsupdates.log存在expire记录。（而不是deadline）

可得直接原因： 系统自动安装补丁后，触发的自动重启。

然而我们的服务器补丁策略应该不是默认安装的，应该是哪里设置错误了。

WSUS服务器已经没有对补丁设置Deadline，服务器默认只能搜索到补丁，并且提示安装，并不是默认安装。

进入服务器运行rsop.msc ，查看更新的组策略跟目标策略不一致。当前策略设置的是默认安装补丁，即是“计划安装”，而不是“提示安装”。

根本原因：服务器设置了错误的OU，导致补丁到服务器中是默认安装补丁。

解决方案

在域控中，将目标服务器转移OU。

后续改造：

  1. 域控的OU策略，统一改成提示安装。
  2. WSUS服务器不设置审批期限。

记一次RD授权故障

Posted on 2020-05-24 Edited on 2024-12-22 In Windows Server

故障现象

Windows Server部署了RD会话的服务，需要指向RD授权服务器，否则只有120天的“宽限期限”。120天之后，是无法直接使用mstsc远程到目标服务器亦或是基于RD会话的服务也将不可用，例如VMware Horizon View的基于“RD会话主机”设立的
RD桌面或者应用将不可用。

原因分析

  1. RDSH资源机没有设置指向RD授权服务器、设置授权模式。
  2. RDSH资源机设置RD授权服务器后没有生效。
  3. RD授权服务器宕机，无法正常提供服务。
  4. RD会话主机角色被禁用。

解决方法

因为License无法使用MSTSC连接，可以使用 mstsc /admin 进行连接。

原因1：

“RDSH资源机没有设置指向RD授权服务器、设置授权模式。”

设置方法：

  1. 进入组策略编辑器 gpedit.msc
  2. 选择 计算机配置 -> 管理模板 -> Windows组件 -> 远程桌面服务 -> 远程桌面会话主机 -> 授权 。
  3. 将“使用指定的远程桌面许可证服务器”，设置为目标RD授权服务器。
  4. 将“设置远程桌面授权模式” ，设置为按用户。
  5. 网络端口禁用。

设置完后需要重启。

原因2：

“RDSH资源机设置RD授权服务器后没有生效。”

我在每台服务器上面运行了注册表后，没有重启服务器。每台服务器虽然不再提示“远程桌面授权模式尚未配置”的告警，并且在RD诊断器中，可以看到已经连接授权服务器，但本质上是没有生效的。

检测 LicenseServer:135 端口是否开放。

原因4：

Linux脚本运行错误

Posted on 2020-05-21 Edited on 2024-12-22 In Linux

故障描述

运行脚本出现：

原因分析

  1. 脚本格式为dos，而Linux只能执行格式为unix的脚本。
  2. 因为在dos/window下按一次回车键实际上输入的是“回车（CR)”和“换行（LF）”，而Linux/unix下按一次回车键只输入“换行（LF）”，所以修改的sh文件在每行都会多了一个CR，所以Linux下运行时就会报错找不到命令。

查看方式：

在vim中打开脚本，输入： set ff ，回车可以看到脚本为dos格式。

解决方法

在vim中打开脚本，输入： set ff=unix 或者 set fileformat=unix ,即可。

什么是叠瓦式硬盘

Posted on 2020-05-16 Edited on 2024-12-22 In 服务器与存储 , 存储

前言

近期想自己组装一台NAS，用于存储自己的冷数据。了解到机械硬盘的坑无处不在，虽然现在硬盘价格便宜，但也需要了解清楚硬盘的差别，避免被割韭菜。

相关信息总结

关于垂直式（PMR）和叠瓦式（SMR）说的是硬盘记录信息的方式。

背景

前提：机械盘读取盘面的磁头跟写入的磁头大小不一致。读磁头的宽度低于写磁头的宽度，因此读写需要的磁道宽度空间并不相同。

PMR方式： 单个保护空间内，仅写入一个磁道。

SMR方式： 单个保护空间内，写入多个磁道。写磁头的宽度要大于读磁头，因此写入前需要将保护空间内的数据提取出来，然后再重复写入，即写入“目标数据”，又再次写入“原有数据”。

看到一个比喻是这么说的：

在单行本里面一行中，写入了2行小字，需要修改某个字时，橡皮檫太大就会擦掉其他的字，因此重新写字时，需要将其他被“误擦”的字给补上。

性能

  1. 读取速度差别不大。
  2. 写入速度，垂直式硬盘要优于叠瓦式硬盘。

“……SMR硬盘对数据量比较大的写数据操作的性能可以达到当前硬盘的写性能，然而小数据量的写操作以及随机写操作的性能很差甚至无法进行。”

分辨

SMR硬盘的共同点：

  1. 小容量、大缓存。（容量大于6T，用256M缓存很正常。）
  2. 名字上带有Archive。
  3. 超大容量。（20T之类的盘，PMR技术还无法实现。）

Vmware Horizon View添加证书

Posted on 2020-05-01 Edited on 2024-12-22

前言

默认情况下，在安装连接服务器或安全服务器时，安装程序会为服务器生成一个自签名证书。但在以下情况中，安装程序仍使用现有证书：

  • 如果 Windows 证书存储区中已存在友好名称为 vdm 的有效证书
  • 如果您从较早版本升级到 Horizon 7，且在 Windows Server 计算机上配置了有效的密钥存储文件，则安装会提取密钥和证书，并将其导入到 Windows 证书存储区中。

生成证书

连接服务器中添加证书

  • 新签发的证书将添加到 Windows 证书存储区内的个人 > 证书文件夹。
  • 对于连接服务器实例或安全服务器，请将证书的友好名称修改为 vdm。

详细操作：

  1. 在 Windows Server 主机上的 MMC 窗口中，展开证书 (本地计算机) 节点并选择个人文件夹。
  2. 在“操作”窗格中，转到更多操作 > 所有任务 > 导入。
  3. 在证书导入向导中，单击下一步并浏览至存储证书的位置。
  4. 选择证书文件并单击打开。
  5. 要显示您的证书文件类型，可以从 文件名下拉菜单中选择其文件格式。
  6. 键入证书文件中所含私钥的密码。
  7. 选择将此密钥标记为可导出。
  8. 选择包含所有已扩展属性。
  9. 单击下一步，然后单击完成。
  10. 新证书会显示在 证书 (本地计算机) > 个人 > 证书文件夹中。
  11. 确认新证书包含私钥。
  12. 在证书 (本地计算机) > 个人 > 证书文件夹中，双击新证书。
  13. 在“证书信息”对话框的“常规”选项卡中，确认显示有以下语句：您有一个与该证书对应的私钥 (You have a private key that corresponds to this certificate)。
  14. 将证书的友好名称改为 vdm。
