记一次vscode更新后的问题

Posted on 2026-05-30 In default

故障现象

VSCode 在一次自动更新后，无法通过vscode连接到linux服务器做一些编辑工作，每次通过vscode连接服务器时，在输入密码后均会卡在 setting up sshd host xxx:downloading vs code server；

原因分析

应该是组件更新后，需要在目标服务器中安装对应插件；由vscode远程连接linux服务器时，服务器会触发wget指令

服务器解析update.code.visualstudio.com域名，得到的IP地址有异常，导致无法正常通讯；

解决方法

在linux服务器中添加 update.code.visualstudio.com 的解析记录至本地 hosts文件；

背景

旁路由

旁路由其实并不是路由，路由是用来连接不同网络的，最常用的就是用来连接互联网和局域网。旁路由起到的主要是网关的作用，是用来分流数据和扩展插件的。因此，严谨一点的叫法应该是 旁路网关，只是大家好像约定俗成了都叫做旁路由，所以我们这里也跟着叫旁路由，但是要明白它的核心是网关而不是路由。

要实现分流的常规做法是改造现有网络架构，修改主路由配置；但“旁路由”模式可以不折腾现有的网络架构，只是在链路上挂载一个组件即可实现网络访问的分流；

常规拓扑如下：


graph LR
Client-->主路由
主路由-->外网

加装旁路由的拓扑如下：


graph LR
Client-->主路由
主路由-->外网
主路由-->旁路由

常规的做法是需要部署DHCP服务器，在DHCP服务器中对特定设备的默认网关做区分，常规设备就使用正常的网关，无法走代理流量；而特殊的设备，默认网关则调整为旁路由的IP；
