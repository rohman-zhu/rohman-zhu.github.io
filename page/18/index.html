ESXi上的虚拟机网络异常

Posted on 2021-08-08 | In VMware
Words count in article: | Reading time ≈

故障现象

虚拟机的操作系统内配置完网络后，部分服务器可以ping通此服务器，且虚拟机内部无法ping通网关。
长ping虚拟机的时间内，只有25%的时间可以ping通此虚拟机。

原因分析

虚拟机使用的端口组异常，上行链路四个网口是做了lacp。而ESXi中的标准交换机，只有1个uplink，其他3个uplink为备用状态。

解决方法

将ESXi的标准交换机的上行链路，4个上行链路都改成Active状态。

Windows Server只识别到一个CPU

Posted on 2021-08-08 | In Windows Server
Words count in article: | Reading time ≈

故障现象

物理服务器有2颗物理CPU，而操作系统只识别到1颗CPU。

使用wmic查询也只有一个CPU、一个逻辑CPU；

显示只有1

原因分析

  1. 硬件故障
  2. 操作系统设置故障

解决方法

原因1，硬件故障

进入服务带外检查，发现CPU状态正常。

原因2，操作系统设置异常

进入msconfig，微软配置项处。

进入引导页面，进入高级选项，里面有“引导高级选项”，在处理器个数这一栏目内，发现数量是1。于是将处理器个数前面的复选框，取消勾选，点击确定后，重启操作系统。
