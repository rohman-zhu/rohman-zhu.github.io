疫情期间VDI小结

Posted on 2022-03-20 Edited on 2024-12-22 In 杂谈

背景

略

分类小结

场景分配

类型 配置 备注
常规用户 4C8G
轻度编译用户 4C16G
重度编译用户 8C16G
超重度编译用户 16C32G

以上4中用户类型中，若资源条件允许，应让“轻度编译用户”类型作为常规用户，当下的个人PC配置环境基本是16G内存起步。如果内存分配过小，用户在实际使用的时候很容易出现“Out of memory”的错误。

以上环境的单核基础频率是2.1GHz，也就是说一个重度用户对CPU的需求为16.8GHz左右。同一个任务下，4C要比8C多耗时40分钟左右。

业务总结

使用场景评估

需要清楚用户的访问链路，思考几个问题：

  1. 用户的访问链路应该是怎么样的？
  2. 是否会与其他的远程工具冲突，如原本在外网就有解析的入口，是否会与VPN冲突？
  3. 私有云的出口网络带宽是否能支撑用户的访问需求？带宽需求大概是多少？
  4. 是否可以清楚监控到对应节点的负载情况？
  5. 用户的核心需求是什么？是远程桌面？还是VDI？

No.5非常重要，避免资源浪费，用户也许不清楚自己需要什么，更多情况下就是什么资源都会申请，这无疑会增加运维人员与支持人员的工作量。

信息安全侧评估

需要明确信息安全红线，确保数据进出方向，对于敏感策略一定要设置有监控。所有的部署方案、防火墙策略申请应有审批报备。

效率总结

管理侧

对于 VMware Horizon View 中的 RDS 桌面部署，基本上都是在Chrome浏览器中用鼠标点击授权，这个重复性的操作可以用Python3+Selenuim+Chrome来做自动化点击部署。

资源侧

对于初始化RDSH服务器，可以使用 Windows + Openssh + Ansible 来批量部署。

RDSH服务器均为Windows服务器，在上面安装Openssh，并配置允许Ansible账户来部署。

Ansible侧则是需要写好相应的脚本批量下发即可，这里要注意Ansible服务器的配置，避免跑的任务太多导致CPU爆满。

基础架构侧

底层资源要充足，在紧急时候能够提供所需资源，这个需要与业务方商谈，让其做相关预算。

机房停电-vSAN集群相关处理想法

Posted on 2022-03-05 Edited on 2024-12-22 In 杂谈

背景

异地的机房停电，无法远程处理服务器、交换机设备问题。现场有IT人员，但是无PC可以操作，并且无相关经验。

一些想法

若当地IT人员无处理经验，则需要远程指导现场人员处理。
从处理方式上面，最好还是从GUI界面操作，其次的方法才是从 ESXi Shell进入。
从运维方式上面，除了增加远端跳板机，还需要增加本地跳板机。

以上两个方法存在的问题：

  • 均需要交付给当地人员管理员账户。【此处存在信安问题，一定要走报备】
  • GUI界面需要接入现场的业务交换机。【此处存在信安问题，并且交换机端需要提前配置】

GUI界面操作方式

  1. 笔记本接入业务交换机，并配置与vCenter同网段的IP。
  2. 进入vCenter，关闭所有虚拟机（除了vCenter）后，将vSAN节点的ESXi进入维护模式。（选择“不迁移数据”）
  3. 所有vSAN节点进入维护模式后，再其关机。
  4. 最后进入带有vCenter的ESXi，关闭vCenter，再将ESXi进入维护模式后关机。
  5. 关机后的服务器，需要将电源移除。

注： 本次场景中vCenter是VCSA。

CLI界面操作方式

  1. 给物理服务器接入显示器，进入ESXi Shell 。
  2. 指令关闭虚拟机，确保所有vSAN节点上的虚拟机关机后，再进入维护模式。
  3. 所有节点进入维护模式后，再关机。
  4. 关机后的服务器，需要将电源移除。

记一次Linux因网络导致宕机的故障

Posted on 2022-01-12 Edited on 2024-12-22 In 服务器与存储

故障现象

  1. 服务器的万兆网卡模块没有发射光；
  2. 操作系统还在运行中；
  3. 重启网卡依旧不能生效；
  4. 需要将服务器关机停5分钟左右，再开机，网卡才能恢复正常；

原因分析

可能是因为网卡流量过大，导致系统故障，网卡不可用。

解决方法

分析该服务器流量较大，从zabbix监控看到该网卡的流量已经跑满10Gb，因此增加多一个网口，将双网口组成LACP。

HPE服务器阵列扩容

Posted on 2022-01-05 Edited on 2024-12-22 In 服务器与存储

背景

当前阵列可用空间已经剩下不多了，需要扩容。已知服务器还有一个硬盘槽位，HPE的阵列卡为P440a。

前置条件

  1. 是否支持Smart Storage Administrator内执行扩容。（可以咨询供应商）
  2. 目标阵列的硬盘是什么类型的，SAS还是SATA？单盘容量是多少？
  3. 是否可以不用停机就执行扩容？（SSA是否可以在操作系统内安装）
  4. 底层完成扩容需要多久？

以上条件均需要评估好。
